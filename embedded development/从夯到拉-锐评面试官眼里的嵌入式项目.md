## 一、面试官关注的核心维度

### 1.1 项目深度 vs 广度

| 维度 | 初级水平 | 中级水平 | 高级水平 |
|------|---------|---------|---------|
| **硬件理解** | 能看懂原理图 | 能独立设计简单电路 | 能进行EMC/信号完整性分析 |
| **软件能力** | 会调用API | 能写驱动层代码 | 能做RTOS移植和优化 |
| **系统思维** | 实现单一功能 | 模块化设计 | 架构设计和性能调优 |

### 1.2 "夯"与"拉"的含义

> [!tip] 核心概念
> - **夯（基础夯实）**：
>   - GPIO、UART、I2C、SPI等基础外设的熟练使用
>   - 中断处理、定时器配置
>   - 基本的调试能力（串口打印、逻辑分析仪）
>
> - **拉（能力拔高）**：
>   - 复杂通信协议的实现（CAN、USB、Ethernet）
>   - RTOS多任务调度和资源管理
>   - 低功耗设计、Bootloader开发
>   - 算法移植和优化（PID、滤波、FFT）

## 二、嵌入式项目必备模块清单

### 2.1 硬件相关（必须掌握）

```
基础外设：
├─ GPIO（输入输出、中断）
├─ UART（串口通信、调试接口）
├─ I2C（传感器、EEPROM）
├─ SPI（Flash、显示屏）
├─ ADC/DAC（数据采集、波形生成）
├─ PWM（电机控制、LED调光）
└─ 定时器（PWM生成、延时、计数）

进阶外设：
├─ CAN总线（工业控制、汽车电子）
├─ USB（设备/host模式）
├─ Ethernet（网络通信）
├─ SDIO（SD卡读写）
└─ DMA（高效数据传输）
```

### 2.2 软件架构层次

```c
// 典型的嵌入式软件分层
┌─────────────────────────────┐
│       应用层 (App Layer)     │  ← 业务逻辑、UI交互
├─────────────────────────────┤
│      中间件层 (Middleware)    │  ← 协议栈、文件系统、GUI
├─────────────────────────────┤
│      驱动层 (HAL/Driver)     │  ← 外设驱动抽象
├─────────────────────────────┤
│      BSP板级支持包            │  ← 硬件初始化、引脚配置
├─────────────────────────────┤
│      硬件抽象层 (Register)   │  ← 寄存器操作
└─────────────────────────────┘
```

## 三、面试高频问题及回答策略

### 3.1 项目经验类

**Q: 请介绍你最满意的嵌入式项目**

> [!warning] 低分回答示例
> ❌ "我做了一个智能小车，用了STM32，能遥控..."

> [!success] 高分回答模板
> ```
> 【项目背景】
> - 目标：实现XXX功能，解决XXX痛点
> - 选型理由：为什么选STM32F103而非其他MCU
> - 技术指标：响应时间<XXms，功耗<XXmA
>
> 【技术难点】
> - 难点1：XXX问题的分析与解决
>   → 方案对比（方案A vs 方案B）
>   → 最终选择及原因
>   → 实测数据验证
>
> - 难点2：XXX性能优化
>   → 优化前：XXms/XXmA
>   → 优化后：XXms/XXmA（提升XX%）
>
> 【个人贡献】
> - 独立完成：驱动层+应用层（占比XX%）
> - 团队协作：与硬件工程师配合解决XXX问题
> - 文档输出：设计文档、测试报告
> ```

### 3.2 技术深度类

**Q: I2C通信遇到ACK失败怎么排查？**

```
排查流程：
1. 硬件层面
   ├─ 检查上拉电阻（通常4.7K~10K）
   ├─ 示波器查看SCL/SDA波形（电平、时序）
   └─ 确认设备地址是否正确

2. 软件层面
   ├─ 检查初始化配置（时钟速度、模式）
   ├─ 排查总线占用（是否被锁死）
   └─ 加入超时机制和错误重试

3. 常见原因
   ├─ 设备未上电/地址错误
   ├─ 总线短路或干扰
   └─ 时序不满足（setup/hold time）
```

### 3.3 系统设计类

**Q: 如何设计一个低功耗的温湿度采集系统？**

```
设计方案：
┌─ 硬件选型
│  ├─ MCU：选择低功耗系列（STM32L0/L4、MSP430）
│  ├─ 传感器：支持休眠模式的数字传感器（SHT30、BME280）
│  └─ 电源管理：LDO效率优化、电池电量监测
│
├─ 软件策略
│  ├─ 休眠机制：Stop Mode + RTC唤醒
│  ├─ 采集频率：按需调整（1次/分 ~ 1次/小时）
│  ├─ 数据缓存：减少Flash写入次数（磨损均衡）
│  └─ 通信优化：批量上传、压缩传输
│
└─ 功耗估算
   ├─ 运行态：XX mA（采集+处理，耗时XX ms）
   ├─ 休眠态：XX μA
   └─ 平均功耗：XX mAh/day（理论续航XX天）
```

## 五、常见项目类型及技术栈推荐

### 5.1 入门级项目（1-2周）

1. **智能交通灯控制系统**
   - 技术：GPIO + 定时器 + 状态机
   - 亮点：按键切换模式、数码管显示倒计时

2. **环境监测终端**
   - 技术：I2C/SPI传感器 + OLED显示 + 串口上报
   - 亮点：阈值报警、数据曲线绘制

3. **简易示波器**
   - 技术：高速ADC + DMA + TFT屏显示
   - 亮点：触发捕获、频率测量

### 5.2 进阶级项目（3-4周）

1. **平衡小车**
   - 技术：MPU6050 + PID算法 + PWM电机控制
   - 亮点：自平衡、蓝牙遥控、参数整定界面

2. **智能网关**
   - 技术：WiFi/LoRa + MQTT + 云平台对接
   - 亮点：多协议转换、OTA远程升级

3. **便携式医疗设备原型**
   - 技术：低功耗设计 + 心率/血氧传感器 + 数据存储
   - 亮点：长时间续航、历史数据回放

### 5.3 高级项目（1-2个月）

1. **工业数据采集器**
   - 技术：多路ADC + CAN总线 + RTOS + 文件系统
   - 亮点：实时采集、本地存储、远程上传

2. **无人机飞控系统**
   - 技术：六轴传感器融合 + PID控制 + GPS导航
   - 亮点：姿态解算、自动巡航、故障保护

## 六、面试避坑指南

### 6.1 绝对不能说的

- ❌ "这个模块是网上复制粘贴的"
- ❌ "我不清楚底层原理，只会用库函数"
- ❌ "项目都是老师/学长完成的"
- ❌ "没有遇到过bug"（太假了！）

### 6.2 正确的回答方式

> [!example] 当被问到不会的问题时
> ✅ "这个问题我目前了解不够深入，
>    但我知道它涉及XXX方面。
>    如果在实际项目中遇到，
>    我会先查阅数据手册/datasheet，
>    再参考开源实现，最后结合实际场景验证。"
>
> ✅ 展现学习能力和解决问题的思路，
>    而非死记硬背的知识点。

### 6.3 准备材料清单

- [ ] **源码仓库**（GitHub/Gitee，要有commit记录）
- [ ] **原理图PDF**（能讲清楚关键电路设计）
- [ ] **演示视频/照片**（实物运行效果）
- [ ] **技术文档**（设计说明、测试报告）
- [ ] **问题记录本**（遇到的坑和解决方案）

## 七、学习路线建议

### 第一阶段：打基础（1-2个月）

```
Day 1-15:   C语言强化（指针、结构体、位操作）
Day 16-30:  单片机基础（GPIO、中断、定时器）
Day 31-45:  通信协议（UART、I2C、SPI、ADC）
Day 46-60:  小项目实战（综合运用以上知识）
```

### 第二阶段：深钻研（2-3个月）

```
Month 1:    RTOS入门（FreeRTOS任务、同步机制）
Month 2:    驱动开发（编写自己的HAL库）
Month 3:    中型项目（加入文件系统、GUI、网络）
```

### 第三阶段：广扩展（持续）

- 学习不同平台（STM32、ESP32、NXP、TI）
- 关注行业动态（汽车电子、IoT、工业控制）
- 参与开源社区（阅读优秀项目源码）
- 刷题巩固（LeetCode嵌入式相关、牛客网）
