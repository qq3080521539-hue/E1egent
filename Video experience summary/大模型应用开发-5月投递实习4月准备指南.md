## 4月准备工作清单

### 第一周：技术基础夯实（第1-7天）

#### 1. 大模型核心概念掌握
**必须掌握的理论知识**：
- Transformer架构基础原理
- LLM的工作机制（预训练、微调、推理）
- Tokenization和上下文窗口概念
- Temperature、Top-P等生成参数含义
- Prompt Engineering基本原则

**推荐学习资源**：
- Andrej Karpathy的"Let's build GPT from scratch"系列
- 吴恩达的Generative AI短课程
- OpenAI官方文档中的Concepts部分

#### 2. 主流框架和工具熟悉
**必学工具链**：
- **LangChain/LangGraph**：LLM应用开发框架
- **OpenAI API / 国内替代方案**：API调用实践
- **Hugging Face Transformers**：模型加载和使用
- **LlamaIndex**：RAG应用开发
- **Vector Database**（Pinecone/Milvus/Chroma）：向量数据库

**动手实践要求**：
```python
# 最小可行项目示例
from langchain.chat_models import ChatOpenAI
from langchain.schema import HumanMessage

llm = ChatOpenAI(temperature=0.7)
response = llm([HumanMessage(content="你好")])
print(response.content)
```

#### 3. 开发环境搭建
- 配置Python 3.9+环境
- 安装核心依赖：`pip install langchain openai chromadb`
- 准备API Key（OpenAI或国内替代）
- 熟悉Jupyter Notebook或VS Code开发

### 第二周：项目实战与作品集（第8-14天）

#### 1. 完成至少2个完整项目

**项目一：智能问答系统（RAG）**
- 功能：基于文档的智能问答
- 技术点：
  - 文档加载与分块（Document Loading & Chunking）
  - Embedding生成与向量存储
  - 相似度检索与答案生成
  - 引用来源展示
- 工时：3-4天
- 代码量：200-300行

**项目二：多Agent协作系统**
- 功能：复杂任务的多角色协作完成
- 技术点：
  - Agent定义与工具调用
  - 任务分解与执行流程
  - 记忆机制实现
  - 结果整合输出
- 工时：3-4天
- 代码量：300-400行

#### 2. 项目文档整理
每个项目需包含：
- README.md（项目介绍、功能演示、技术架构）
- 核心代码注释清晰
- 运行说明和环境配置
- 效果截图或Demo视频链接

#### 3. GitHub仓库优化
- 创建专业的GitHub Profile
- 项目代码结构清晰
- 添加LICENSE文件
- 编写详细的commit message

### 第三周：面试准备与材料完善（第15-21天）

#### 1. 简历优化
**简历必备内容**：
- **教育背景**：学校、专业、GPA（如优秀）
- **技术技能**：
  - 编程语言：Python（熟练）、JavaScript/TypeScript（了解）
  - 框架工具：LangChain、OpenAI API、FastAPI
  - 数据库：PostgreSQL、Redis、向量数据库
  - 其他：Git、Docker、Linux基础
- **项目经历**：
  - 2-3个LLM相关项目（重点突出）
  - 使用STAR法则描述：情境、任务、行动、成果
  - 量化成果："处理XX文档"、"准确率提升XX%"
- **奖项荣誉**：竞赛获奖、奖学金等

**简历格式建议**：
- 控制在1页A4纸
- 使用简洁专业的模板
- 重点内容加粗或使用图标
- 导出PDF格式

#### 2. 算法与编程能力
**必刷题目类型**：
- LeetCode Hot 100中的中等难度题
- 重点关注：字符串、数组、哈希表、DFS/BFS
- 目标：能独立完成Medium难度题目

**每日练习计划**：
- 每天2-3道算法题
- 重点复习：双指针、滑动窗口、动态规划基础

#### 3. 面试高频问题准备

**技术类问题**：
1. **Transformer原理**：
   - Self-Attention计算过程
   - Positional Encoding作用
   - Multi-Head Attention优势

2. **RAG系统设计**：
   - 文档分块策略选择
   - Retrieval方法对比（向量检索 vs 关键词检索）
   - Reranking的作用和实现

3. **Prompt Engineering技巧**：
   - Few-shot vs Zero-shot
   - Chain-of-Thought (CoT)原理
   - System Prompt设计原则

4. **Agent架构**：
   - ReAct模式
   - Tool Use实现方式
   - Memory机制（短期/长期记忆）

**行为类问题（STAR法则）**：
- "介绍一下你最有挑战性的项目"
- "遇到技术难题如何解决"
- "团队合作中如何沟通协调"
- "为什么选择这个方向/公司"

#### 4. 模拟面试
- 找同学进行mock interview
- 录制自己的回答并回看改进
- 练习用简洁的语言解释复杂概念

## 常见误区与避坑指南

### ❌ 错误做法
1. **只看不练**：看了很多教程但没亲手写代码
2. **项目太简单**：只是调用API没有深入理解
3. **简历注水**：夸大项目难度和个人贡献
4. **忽视算法**：认为AI岗不考算法
5. **准备太晚**：5月才开始匆忙准备

### ✅ 正确做法
1. **项目驱动学习**：边做项目边补充理论知识
2. **深度理解原理**：不仅会用还要懂底层逻辑
3. **量化成果**：用数据说话（性能指标、用户数等）
4. **持续刷题**：每天保持手感
5. **提前准备**：至少提前2-3个月系统准备
